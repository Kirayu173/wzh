# 阶段五关键待决策点（建议版）

1) 二维码内容
- 推荐：traceCode（短码）
- 可选：traceUrl（完整 URL）
- 理由：短码二维码更短、识别率更高、解析逻辑更简单；完整 URL 便于第三方扫码直达页面，但二维码长度/容错率受影响。

2) 溯源详情接口形态
- 推荐：单接口聚合返回（批次 + 物流节点一次返回）
- 可选：分接口（批次与物流节点分别查询）
- 理由：聚合接口可减少客户端请求次数与串联失败点，首屏展示更快；分接口更灵活但会增加前端联调与缓存复杂度。

3) 扫码记录存储
- 推荐：本地 Room 记录
- 可选：服务端记录（需新增接口与隐私告知）
- 理由：本地实现成本低、无额外合规负担；服务端记录可支持多端同步，但需新增接口与数据合规流程。

> 备注：上述方案为推荐选项，若变更需同步调整后端接口、客户端扫码解析与数据存储逻辑。
