# 阶段二关键待决策点（建议版）

1) Token 方案
- 推荐：JWT
- 理由：无状态、易扩展；与 Android 拦截器/过期处理对接直观；便于后续权限与网关扩展。

2) 密码加密算法与强度
- 推荐：BCrypt（成本参数需本地压测确认）【待查证】
- 理由：行业通用、抗暴力破解强；Spring Security 支持完善。

3) 登录态存储位置
- 推荐：Android 使用 SP 保存 Token + 基础用户信息
- 理由：实现成本低、与现有 SPUtils 复用一致；满足阶段二最小闭环。

> 备注：上述方案为推荐选项，若变更需同步调整后端安全配置与客户端拦截/过期处理逻辑。
