# 阶段一关键待决策点（已采用推荐选项）

1) 架构选型
- 采用：MVP
- 理由：复用 `items/shop-mall-android` 的基类成本最低，最快达成“可编译可运行”。

2) JSON 与网络栈
- 采用：Fastjson（与 Retrofit 组合）
- 理由：与 `items/shop-mall-android` 的 Converter/BaseResponse 保持一致，复用成本低。

3) ZXing 依赖与 minSdk 21 兼容方案
- 采用：降级 `zxing:core` 到 3.3.0（与 `com.journeyapps:zxing-android-embedded:4.3.0` 配套）
- 理由：改动少、配置简单，优先保证阶段一编译与运行。
- 备注：Gradle 具体配置与版本需按 `items/zxing-android-embedded/README.md` 再核对【待查证】。
