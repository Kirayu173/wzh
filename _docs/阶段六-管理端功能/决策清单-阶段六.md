# 阶段六关键决策点（已确认）

1) 管理端形态
- 推荐（已选）：新建 Web 后端管理平台
- 可选：App 内隐藏入口（个人中心版本号连点）
- 理由：桌面端操作效率更高、权限隔离更清晰；但需新增前端工程与部署链路
- 备注：Web 技术栈与部署方式【待查证】
2) 商品图片来源
- 推荐（已选）：录入图片 URL
- 可选：本地上传 + 对象存储（需新增上传接口与存储配置【待查证】）
- 理由：避免上传链路与权限配置复杂度，复用现有 `cover_url` 字段
3) 订单管理操作范围
- 推荐（已选）：仅开放发货（`PAID` → `SHIPPED`）
- 可选：管理员手动调整任意状态
- 理由：与现有状态机一致，降低误操作与数据不一致风险
4) 溯源批次更新策略
- 推荐（已选）：全量更新（PUT）
- 可选：局部更新（PATCH）
- 理由：表单实现与后端校验更简单，减少字段合并逻辑
5) 二维码交付方式
- 推荐（已选）：后端返回 PNG 二进制
- 可选：返回图片 URL（需存储与生命周期管理【待查证】）
- 理由：避免额外存储成本，复用现有二维码接口
6) 权限控制策略
- 推荐（已选）：后端强校验 admin + 前端提示拦截
- 可选：仅前端隐藏入口
- 理由：安全底线必须在后端，前端仅提升体验
