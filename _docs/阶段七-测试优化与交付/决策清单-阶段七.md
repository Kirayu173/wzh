# 阶段七关键待决策点（建议版）

1) 自动化测试层级
- 推荐：IT/UT 为主，UI/E2E 仅覆盖订购链路与溯源链路关键路径
- 可选：引入 Espresso/UIAutomator 完整覆盖
- 理由：IT/UT 维护成本较低，回归效率高；UI/E2E 可提升端到端保障，但会增加构建配置与维护成本。

2) 性能基线口径
- 推荐：以“首页/商品详情/订单列表”加载与接口 P95 作为基线
- 可选：扩展到更多页面与 P99 指标
- 理由：核心链路的 P95 更能反映用户体验；扩展口径可提升覆盖，但需更多设备与网络数据。

3) 交付包形式
- 推荐：Release APK + 可执行 JAR
- 可选：仅提供 Debug APK
- 理由：Release 包更贴近验收演示场景，稳定性与可演示性更好；Debug 包仅适合开发联调。

> 备注：若选择可选方案，需同步调整测试范围、演示脚本与验收说明。
